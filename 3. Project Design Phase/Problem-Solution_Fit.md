# Problem-Solution Fit Canvas — EduGenie AI Assistant

| # | Section | Details |
|---|---|---|
| 1 | **Customer Segment(s) [CS]** | University students, self-learners, and educators who study/teach outside formal classroom tools |
| 2 | **Problems / Pains + Frequency [PR]** | Information overload from dense text (daily); no instant way to self-test (per study session); cloud-AI cost/latency for small tasks (frequent, low-value asks) |
| 3 | **Triggers to Act [TR]** | Upcoming exam, assigned reading, need to self-assess before a test, unreliable internet during a study block |
| 4 | **Emotions Before / After [EM]** | Before: overwhelmed, anxious about time and cost. After: confident, in control of a structured study workflow |
| 5 | **Available Solutions [AS]** | Generic chatbots (ChatGPT, Gemini web) — good reasoning but no built-in quiz/roadmap workflow, always cloud-dependent, no offline fallback |
| 6 | **Customer Limitations [CL]** | Budget (can't pay for multiple subscriptions), device (mixed CPU-only laptops), inconsistent internet access |
| 7 | **Behavior + Intensity [BE]** | Repeated tab-switching between separate tools for explaining, summarizing, and quizzing — high daily intensity during exam periods |
| 8 | **Channels of Behavior [CH]** | Online: browser-based study apps, cloud chatbots. Offline: printed notes, manual flashcards |
| 9 | **Problem Root Cause [RC]** | Study tools are single-purpose and 100% cloud-dependent, forcing fragmented workflows and unavoidable per-request API costs |
| 10 | **Your Solution [SL]** | EduGenie: one workbook combining a free local model for lightweight explanations with a cloud model for higher-value reasoning tasks (Q&A, summary, quiz, roadmap) — unifying the full study loop in a single secure, resilient interface |
